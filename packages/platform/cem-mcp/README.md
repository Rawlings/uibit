# Custom Elements Manifest (CEM) MCP Server

[![NPM Version](https://img.shields.io/npm/v/@uibit/cem-mcp.svg?style=flat-square&color=black)](https://www.npmjs.com/package/@uibit/cem-mcp)

A local Model Context Protocol (MCP) server that exposes Custom Elements Manifest (CEM) data of web components to AI assistants and IDE code agents.

This tool works with any standard `custom-elements.json` manifest generated by the Custom Elements Manifest Analyzer.

## Features

- **Recursive Scanning:** Crawls the current working directory recursively for `custom-elements.json` files by default, or scans a path specified via CLI arguments.
- **Context-Window Optimized:** Automatically filters out AST noise, internal module resolutions, and private/protected members (e.g. methods and variables prefixed with `_`), keeping the payload lightweight.
- **Markdown Projections:** Exposes component APIs as clean, readable, token-efficient Markdown API reference sheets as well as JSON.
- **Enhanced Synergy with Extended CEMs:** While fully compatible with any standard CEM, it extracts additional rich metadata if available (e.g. custom CSS states `@cssstate`, lifecycle metadata `@since`, `@deprecated`, `@see`, and detailed method parameters/returns generated by processors like `@uibit/cem-extended`).

## Extended Metadata Support

For even richer AI assistance, combine this MCP server with [@uibit/cem-extended](/tooling/cem-extended). 

Standard Custom Elements Manifests document fields, properties, slots, and events. Integrating `@uibit/cem-extended` enriches the manifest output with:
- **Custom CSS States (`@cssstate`)**: Instructs AI agents on what state selectors are available (e.g., `:hover`, `:active`, `:playing`).
- **Method parameter & return documentation**: Exposes detailed method JSDocs, parameter explanations, and return types to the model.
- **API Lifecycle markers**: Includes annotations like `@deprecated`, `@since`, and `@see` to help agents suggest up-to-date code patterns and avoid deprecated APIs.

## Tools Exposed

1. `list_components`: Lists all resolved custom elements, their packages, and brief descriptions.
2. `get_component_api`: Fetches properties, attributes, slots, events, CSS custom properties, custom CSS states (`@cssstate`), methods, and JSDoc annotations for a specific component.

## Resources Exposed

- `cem-mcp://components/{tag-name}/api.json`: Exposes the filtered JSON API declaration.
- `cem-mcp://components/{tag-name}/api.md`: Exposes a clean, token-efficient Markdown specification.

## Installation & Setup

1. Build the package:
   ```bash
   pnpm run build
   ```

2. Run the server locally via stdio, optionally specifying the directory containing the custom elements manifests to scan:
   ```bash
   node dist/index.js [path-to-scan]
   ```
   If no path is specified, it recursively scans the current working directory (`process.cwd()`), ignoring `node_modules`, `.git`, `.turbo`, and build output folders.

## IDE & Client Configuration

Add this configuration to your MCP-compatible IDE or client (e.g. VS Code Cline/Roo Code, Cursor, or Claude Desktop):

### Running via npx (Recommended)

```json
{
  "mcpServers": {
    "cem-mcp": {
      "command": "npx",
      "args": [
        "-y",
        "@uibit/cem-mcp",
        "/path/to/your/web-components-project"
      ]
    }
  }
}
```

### Running Locally from Source

If you want to run it directly from a local build:

```json
{
  "mcpServers": {
    "cem-mcp": {
      "command": "node",
      "args": [
        "/path/to/uibit/packages/platform/cem-mcp/dist/index.js",
        "/path/to/your/web-components-project"
      ]
    }
  }
}
```
