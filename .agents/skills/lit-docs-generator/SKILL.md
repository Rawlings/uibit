# Lit Docs Generator

Generates API documentation, usage guides, and React documentation pages for the UIBit docs website.

## What It Does

This skill analyzes a component package and generates/updates:

- **Package README.md:** Basic metadata, installation instructions, basic HTML/React usage, accessibility support list, and main docs site link.
- **React Docs Page (`apps/docs/src/pages/components/<id>/index.tsx`):** Renders a live demonstration block, imports/raw-loads examples, documents accessibility requirements, features list, and imports the component's `custom-elements.json` manifest.
- **Example Files (`apps/docs/src/pages/components/<id>/examples/`):** React functional components showing isolated features or custom setups.
- **JSDoc Annotation Review:** Ensures properties, events, slots, CSS properties (`@cssprop`), and CSS shadow parts (`@csspart`) are correctly annotated in `src/<component-name>.ts` so they populate the dynamically-rendered API tables in the docs app.

## Input Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `componentPath` | string | ✓ | Path to component package (e.g., "components/scroll-progress") |

## Usage Examples

### Generate Component Docs & React Pages
```bash
agent --skill lit-docs-generator --args '{
  "componentPath": "components/scroll-progress"
}'
```

## Documentation Architecture

UIBit does NOT use Storybook. The documentation site is a React single-page application under `apps/docs`. 

Instead of writing static markdown tables for APIs, the docs app imports `custom-elements.json` (the Custom Elements Manifest generated by `cem analyze` during package builds) and dynamically renders tables for properties, events, methods, slots, CSS custom properties, and CSS parts.

Therefore, the documentation generation process must follow this flow:

### 1. Annotate Component TS Source (JSDoc)
To ensure the dynamic API tables populate correctly, you must verify the component has accurate JSDoc descriptions and annotations:

```typescript
/**
 * Thin progress bar that tracks scroll position.
 *
 * @cssprop [--uibit-scroll-progress-height=0.1875rem] - Height of the progress bar
 * @cssprop [--uibit-scroll-progress-bg=transparent] - Background track color
 * @cssprop [--uibit-scroll-progress-color=#000000] - Fill color of the indicator
 * 
 * @csspart [progress] - Element representing the filling progress bar
 */
@customElement('uibit-scroll-progress')
export class ScrollProgress extends UIBitElement {
  /** CSS selector of a custom scrollable container to track. Defaults to window. */
  @property({ type: String, attribute: 'target-selector' }) targetSelector?: string;
  
  // ...
}
```

### 2. Generate Package README.md (`components/<id>/README.md`)
Create a brief overview README:
- Describe the component.
- Installation scripts (`pnpm add @uibit/<id>`).
- Simple usage examples for HTML and React.
- Accessibility compliance overview.

### 3. Generate Docs React Page (`apps/docs/src/pages/components/<id>/index.tsx`)
Create a React module that defines the component docs details. This file must export a `ComponentDocData` configuration:

```tsx
import '@uibit/scroll-progress'; // Import component so it registers in the DOM
import manifest from '@uibit/scroll-progress/custom-elements.json'; // Dynamic API source
import { ComponentDocData } from '../../../types/docs';
import containerExample from './examples/scroll-container';
import containerRaw from './examples/scroll-container?raw'; // Load raw file string for code panel

function ScrollProgressDemo() {
  return (
    <div id="demo-box" className="relative h-64 overflow-y-scroll border rounded-lg">
      <uibit-scroll-progress
        target-selector="#demo-box"
        class="sticky top-0 z-10 block"
      ></uibit-scroll-progress>
      <div className="p-4 space-y-4">
        {/* scrollable text */}
      </div>
    </div>
  );
}

const data: ComponentDocData = {
  manifest,
  Demo: ScrollProgressDemo,
  demoCode: {
    html: `<uibit-scroll-progress target-selector="#scroll-box"></uibit-scroll-progress>`,
    react: `<uibit-scroll-progress target-selector="#scroll-box" class="sticky top-0" />`,
  },
  examples: [
    { ...containerExample, code: { react: containerRaw } }
  ],
  a11y: {
    wcagLevel: 'AA',
    requirements: [
      'Renders with role="progressbar", aria-valuemin="0", and aria-valuemax="100".',
      'The progress bar is decorative and does not intercept focus or keyboard events.'
    ]
  },
  features: [
    'Tracks standard viewport or custom elements.',
    'Prevents event listener leaks on unmount.'
  ]
};

export default data;
```

### 4. Create Example Files (`apps/docs/src/pages/components/<id>/examples/`)
Create isolated example files (e.g. `scroll-container.tsx`):

```tsx
import '@uibit/scroll-progress';
import { UsageExample } from '../../../../types/docs';

function ScrollContainerDemo() {
  return (
    <div id="container-box" className="h-48 overflow-y-scroll">
      <uibit-scroll-progress target-selector="#container-box" class="sticky top-0" />
      <div className="p-4">Content</div>
    </div>
  );
}

const example: UsageExample = {
  title: 'Custom Scroll Container',
  description: 'Bind the progress tracker to any overflow-scroll container.',
  Demo: ScrollContainerDemo
};

export default example;
```

### 5. JSX Code Formatting Rule
When writing React documentation code:
- Wrapper elements use standard React **`className`** (e.g. `<div className="wrapper">`).
- Web components (`<uibit-*>`) use HTML **`class`** (e.g. `<uibit-scroll-progress class="..." />`).

## Verifying Documentation

To test and preview documentation changes:

1. Compile the component: `pnpm -F @uibit/<component-id> run build`
2. Start the documentation website: `pnpm dev`
3. Open your browser to `http://localhost:5173/<component-id>` and review API tables and interactive demos.
